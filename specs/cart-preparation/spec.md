# Feature Specification: cart-preparation

**Feature**: `cart-preparation`  
**Status**: active  
**Parent Baseline**: `specs/hill-commerce-mvp/spec.md`

## Purpose

定义登录用户在下单前的购物准备链路，覆盖购物车增删改查、地址管理、默认地址规则、结算前勾选与汇总视图，以及异常识别规则，为后续订单创建提供稳定输入。

## Scope

### In Scope

- 登录用户购物车增删改查
- 购物车条目勾选与取消勾选
- 购物车汇总展示
- 地址增删改查
- 默认地址规则
- 结算前独立汇总视图
- 购物车异常识别与提示

### Out of Scope

- 匿名购物车
- 登录后购物车合并
- 真正创建订单
- 优惠券、折扣和复杂价格计算
- 自动清理异常购物车项
- 自动纠正库存问题

## User Journeys

### Journey 1: 加购进入购物车

登录用户可从商品详情页选择 SKU / 数量后加入购物车，并在购物车页中看到对应条目。

### Journey 2: 购物车管理

登录用户可在购物车页修改数量、删除条目、勾选或取消勾选条目，并看到汇总结果变化。

### Journey 3: 地址管理

登录用户可新增、编辑、删除收货地址，并在存在地址时始终维护且仅维护一个默认地址。

### Journey 4: 结算前汇总

登录用户可从购物车进入独立汇总页，查看已勾选条目、默认地址、金额汇总和异常提示；本阶段不真正创建订单。

## Cart Rules

- 购物车只支持登录用户
- 同一用户购物车内，同一 `sku_id` 只允许存在一条记录
- 用户可新增、修改数量、删除、勾选、取消勾选购物车项
- 购物车项数量必须为正整数
- 购物车页汇总金额仅基于当前勾选项计算
- 购物车条目最少应展示商品名称、SKU 规格文本、销售价格、数量、小计和状态提示

## Address Rules

- 地址管理只对登录用户开放
- 用户可新增、编辑、删除收货地址
- 只要用户存在地址，就必须且只能有一个默认地址
- 新增首个地址时，该地址自动成为默认地址
- 设置新的默认地址时，原默认地址自动取消默认状态
- 删除默认地址时，若仍有其他地址，系统必须自动补出新的默认地址
- 地址最少应包含收货人、联系电话、所在地区、详细地址和默认地址标识

## Pre-Checkout Summary Rules

- 购物车页负责勾选与初步汇总
- 用户点击“去结算”后进入独立汇总页
- 汇总页只展示当前已勾选购物车项
- 汇总页展示默认地址作为首选收货地址
- 汇总页最少应展示商品名称、SKU 规格、单价、数量、小计、合计和异常状态
- 汇总页不创建订单，只负责确认当前输入条件是否具备进入下单阶段的资格
- 若无有效地址或勾选项存在异常，汇总页必须阻止继续进入后续下单流程

## Exception Recognition Rules

系统必须识别并提示以下异常：

- 商品已下架
- SKU 已失效
- SKU 已禁用
- 库存不足
- 地址缺失或默认地址不可用

对应处理规则：

- 异常购物车项必须保留，不自动删除
- 异常项在购物车页和汇总页都必须有明确状态提示
- 异常项不得被视为有效可结算项
- 若用户勾选异常项，汇总页必须明确指出异常原因
- 若所有勾选项都异常，汇总页必须明确显示当前无可结算商品
- 异常识别属于下单前预校验，最终有效性仍以后续订单创建阶段服务端校验为准

## Business Rules

- 购物车和地址管理仅对登录用户开放
- 不支持匿名购物车
- 不支持登录后购物车合并
- 购物准备链路依赖 `product-discovery` 已定义的商品详情和前台可见性规则
- 购物准备链路在“汇总确认但未创建订单”处结束
- 若用户存在地址，则系统中始终必须存在且仅存在一个默认地址

## Acceptance Criteria

- 登录用户可完成加购、修改数量、删除购物车项
- 登录用户可完成地址新增、编辑、删除，并维护默认地址
- 用户可在购物车页勾选条目并进入独立汇总页
- 汇总页可展示默认地址、已勾选条目和金额汇总
- 汇总页可识别并提示失效商品、失效 SKU、SKU 禁用、库存不足等异常
- 存在异常项或无有效地址时，系统不会把用户视为已具备进入真正下单流程的条件

## Boundaries and Dependencies

- 本 feature 仅负责购物准备，不负责真正创建订单
- 商品发现、商品详情和前台可见性规则由 `product-discovery` 定义
- 登录态与权限事实源由 `auth-permission` 定义
- 真正下单、订单快照和最终校验由 `order-checkout` 定义
