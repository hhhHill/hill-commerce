# Cart Preparation Manual Verification

## Scope

验证 `cart-preparation` 的登录用户购物准备链路：

- 商品详情页加购
- 购物车增删改查与勾选
- 地址新增、编辑、删除、默认地址维护
- 结算前汇总页
- 异常识别与阻断

## Preconditions

- 后端与前端本地服务已启动
- 数据库中至少存在一个前台可见、SKU 启用且有库存的商品
- 准备一个 `CUSTOMER` 账号和一个 `SALES` 账号

## Happy Path

1. 使用 `CUSTOMER` 账号登录前台。
2. 进入任一商品详情页，确认可看到 SKU 选项、数量控制和“加入购物车”按钮。
3. 选择一个有库存的 SKU，加入购物车。
4. 打开 `/cart`，确认新增条目出现，数量、价格、小计正确。
5. 在购物车中修改数量，确认页面刷新后数量和汇总金额同步变化。
6. 取消勾选该条目，确认购物车汇总中的勾选数量与金额归零。
7. 重新勾选该条目，确认汇总恢复。
8. 进入 `/account/addresses`，新增首条地址，确认其自动成为默认地址。
9. 新增第二条地址，确认第二条默认不是默认地址。
10. 将第二条地址设为默认地址，确认地址列表中仅第二条带有默认标记。
11. 返回 `/cart`，点击“去结算前汇总”进入 `/checkout-summary`。
12. 确认汇总页只显示已勾选条目，且默认地址正确展示。
13. 确认汇总页显示“已具备进入下单阶段的前置条件”或等价文案。

## Address Edge Cases

1. 删除当前默认地址，确认系统自动将剩余地址中的一条设为默认地址。
2. 删除最后一条地址，确认地址列表回到空状态。
3. 返回 `/checkout-summary`，确认页面提示缺少默认地址，并阻止继续。

## Cart Edge Cases

1. 将同一 SKU 从商品详情页重复加入购物车两次，确认购物车中只保留一条记录，数量为累加结果。
2. 在购物车中删除某条商品，确认条目消失且汇总同步更新。
3. 在购物车中将所有条目取消勾选，进入 `/checkout-summary`，确认显示无勾选商品或等价阻断结果。

## Summary Anomaly Checks

1. 购物车中保留一条已勾选商品。
2. 使用 `SALES` 账号在后台将该商品改为下架，或将对应 SKU 改为禁用，或将库存调低至小于购物车数量。
3. 返回 `CUSTOMER` 的 `/checkout-summary`。
4. 确认该条目显示明确异常原因：
   - 商品下架
   - SKU 已失效 / 已禁用
   - 库存不足
5. 确认汇总页显示当前不可继续，并阻止进入真正下单阶段。

## Expected Result

- 登录用户可完成加购、购物车管理、地址管理和结算前汇总查看
- 默认地址规则始终成立：存在地址时必须且只能有一条默认地址
- 汇总页能识别关键异常并阻止继续进入下单阶段
