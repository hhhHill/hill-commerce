# Feature Specification: checkout-flow-redesign

**Feature**: `checkout-flow-redesign`
**Status**: active

## Purpose

将购物车→确认订单→支付→下单结果的前台购物流程，从当前的"左右分栏+大圆角卡片"风格改造为淘宝风格的"单列紧凑列表+底部固定结算栏"体验。

参考方向：淘宝 App 下单流程。

## Scope

### In Scope

- 购物车页布局重写：单列列表+底部固定结算栏
- 确认订单页重写：合并原 checkout-summary + checkout，单列紧凑布局
- 支付页去卡片化：统一淘宝风格
- 订单结果页去卡片化
- 删除 `/checkout-summary` 独立页面及其组件
- 异常识别逻辑从独立汇总页迁移到购物车页行内提示

### Out of Scope

- 后端 API 变更
- 支付逻辑变更
- 地址管理页面
- 商品详情页
- 首页/分类页
- 订单中心/订单详情页

## Flow

```
购物车 → 确认订单 → 支付页 → 下单结果
  ↑        ↑          ↑
 异常提示  地址确认   模拟支付
 行内标记  商品清单   成功/失败
```

原 `/checkout-summary` 页面删除，其异常预校验逻辑合并到购物车页（勾选异常项时行内红色提示），地址确认合并到确认订单页顶部。

## Visual Language

| 元素 | 之前 | 之后 |
|------|------|------|
| 页面底色 | `#f5f5f5` | `#ffffff` |
| 商品容器 | `surface-card rounded-lg` 卡片 | 无包裹，分隔线区分 |
| 圆角 | 8-30px 卡片圆角 | 无卡片圆角，按钮 `rounded-full` |
| 阴影 | `shadow-[0_18px_50px_rgba(...)]` | 无阴影 |
| 价格汇总 | 右侧 sidebar | 底部内联 + 固定底部栏 |
| 地址展示 | `rounded-[28px]` 大卡片 | 紧凑一行，右箭头可点击 |
| 间距 | `gap-4`/`gap-6` | `gap-2`/`gap-3` |
| 边框 | `border border-black/10` | `border-b border-[#f0f0f0]` |

## Page Specs

### 1. Cart `/cart`

全白底单列布局：

```
┌─────────────────────────────────┐
│ 导航栏：返回 + 购物车标题 + 编辑 │
├─────────────────────────────────┤
│ ○ 全选                          │
├─────────────────────────────────┤
│ ○ [60×60] 商品名称(2行省略)      │
│          颜色：白；尺码：M        │
│          ¥99.00       [−] 1 [+] │
│          异常：红色文字行内提示    │
├─────────────────────────────────┤
│ ○ [60×60] 商品名称2              │
│          ...                    │
├─────────────────────────────────┤
│ ██████ 底部固定栏 ██████████████ │
│ ○ 全选  合计：¥158.00  [结算(2)] │
│         已选2件，不含运费         │
└─────────────────────────────────┘
```

- 每件商品不再用 `surface-card` 包裹，改为纯列表行
- 勾选圈替代 checkbox，选中为橙色实心圆
- 数量加减从 `rounded-full` 改为方形小按钮
- 下架/SKU失效/库存不足：行内红色文字标记，不显示勾选圈，不可勾选
- 删除操作通过左滑或行末删除图标
- 空状态：居中提示+去逛逛按钮
- 底部栏仅在至少勾选 1 件时显示结算按钮

```
├─────────────────────────────────┤

│ 广东省广州市天河区xxx路xxx号      │
├─────────────────────────────────┤
│ 店铺名称                        │
│ [60×60] 商品名称(2行)    ¥99.00 │
│         颜色：白；尺码：M   ×1   │
├─────────────────────────────────┤
│ 商品总额                ¥99.00  │
│ 运费                    ¥0.00   │
│ 合计                    ¥99.00  │
├─────────────────────────────────┤
│ ██████ 底部固定栏 ██████████████ │
│ 合计：¥99.00        [提交订单]    │
└─────────────────────────────────┘
```

- 点击地址行弹出地址选择（弹窗或底部 sheet）
- 商品不包裹卡片，用分隔线区分店铺
- 无地址时地址行显示红色提示"请添加收货地址"
- 提交失败（库存不足等）在按钮上方显示红色错误文字
- 提交中按钮显示 loading 状态

## Acceptance Criteria

- 购物车页单列列表，无卡片包裹，底部固定结算栏
- 购物车异常商品行内红色提示，不可勾选
- 确认订单页合并原汇总+确认功能，单列布局
- 确认订单页地址紧凑一行，点击可切换
- 支付页去掉大圆角卡片和阴影
- 订单结果页精简布局，去掉大圆角卡片
- 删除 `/checkout-summary` 路由
- 三个页面统一白底+分隔线+底部固定栏风格
- 移动端和桌面端均可正常使用
- 现有业务逻辑不变（加购、下单、支付流程不受影响）

## Boundaries and Dependencies

- 本 feature 仅修改前端 UI 层，不改变任何 API 契约
- 异常识别规则由 `cart/spec.md` 定义
- 支付流程由 `payment/spec.md` 定义
- 全局视觉令牌由 `storefront/ux-transformation/spec.md` 定义
