# Feature Specification: ecommerce-ux-transformation

**Feature**: `ecommerce-ux-transformation`
**Status**: active

## Purpose

将商城前端从当前「精品店/暖棕米色」视觉风格，改造为「中国高转化运营型电商」风格。

参考方向：
- 抖音商城（主要参考）：白底 + 渐变暖橙 + 强对比
- 拼多多 / Temu：高信息密度、强运营感

## Design Goals

- 提高点击率、加购率、下单率、用户停留时间
- 增强「逛」的感觉和购买冲动
- 在现有 API 数据约束内完成改造，不依赖新的后端功能

## Approved Refinement: Taobao-like Homepage Feed

本轮首页 refinement 的批准方向为「淘宝首页 Feed 化」，但不 1:1 复制淘宝，仅学习其电商设计语言：

- 高密度商品流优先，而非后台管理系统式卡片陈列
- 强搜索、强运营 Banner、强价格刺激、强促销标签
- 桌面端使用左侧固定分类栏 + 右侧运营内容与商品流
- 移动端保持双列商品流，分类入口改为顶部横向滑动
- 商品卡以图片、价格、标签、销量、店铺名建立点击欲望

### Fixed Homepage Categories

首页分类入口必须使用固定运营分类，保证前台导航稳定，不受后台分类命名波动影响：

1. 手机数码
2. 家用电器
3. 服饰鞋包
4. 美妆个护
5. 家居生活
6. 食品饮料
7. 母婴玩具
8. 运动户外
9. 汽车用品
10. 其他分类

分类映射规则：

- 若后端返回分类名称与固定分类完全匹配，则该入口链接到对应后端分类页
- 若后端存在不在固定分类范围内的分类，则统一归入「其他分类」
- 若固定分类暂时没有后端对应分类，首页仍展示该入口，但可链接到搜索页或分类聚合页
- 此规则仅约束首页前台导航，不改变后端分类模型

## Scope

### In Scope

- 全局设计令牌（CSS 变量）：色彩、字体、间距、圆角、阴影
- 商品卡组件视觉改造
- 商品列表栅格调整
- 搜索栏视觉改造
- CTA 按钮体系统一
- 首页布局重组（Landing Page → 商品 Feed）
- 商品详情页布局重组（双栏 → 单列移动优先）
- 购物车页底部固定结算栏
- 移动端底部导航栏（新增组件）
- 其他前台页面色彩/字体统一
- Admin 后台页面：保持高对比度、高信息密度，不做暖色运营风格改造
- 首页固定分类入口：桌面左侧固定栏，移动端横向滑动
- 首页运营 Banner 与「限时秒杀」「百亿补贴」「猜你喜欢」区块标题
- 商品卡前端派生展示字段：划线原价、销量、店铺名、促销标签

### Out of Scope

- 任何后端 API 变更
- 真实划线原价、券后价、已售数量、评分、运营标签的数据源接入（本轮仅做前端派生展示字段）
- 无限滚动（需后续前端实现）
- 促销倒计时、满减提示（需后端支持）
- 个性化推荐（需推荐 API）
- 优惠券系统（需后端 coupon 模块）

## Component Specs

### 商品卡 ProductCard

**文件**: `frontend/next-app/src/features/storefront/catalog/product-card.tsx`

基于现有 API 数据（`id`, `name`, `salePrice`, `coverImageUrl`）和前端派生展示字段：

- `originalPrice`: 根据 `salePrice` 稳定派生，仅用于视觉展示
- `sales`: 根据 `id` 稳定派生，例如「已售 2w+」
- `shopName`: 根据 `categoryId` / `id` 稳定派生，例如「官方旗舰店」
- `tags`: 根据 `id` 稳定派生，例如「百亿补贴」「包邮」「官方」「自营」「限时」

这些派生字段不得写回后端，也不得作为业务计算依据。

```
┌──────────────────────┐
│                      │
│     商品图片          │  占卡片高度约 70%-75%，1:1 或 4:5
│                      │
│  [百亿补贴][包邮]     │  促销标签，4px 圆角，高密度
│  商品名称（最多2行）   │  13px, line-clamp-2
│  ¥ 19.90  ¥39.90     │  价格第一视觉，原价划线
│  已售2w+ 官方旗舰店   │  销量 + 店铺名
└──────────────────────┘
```

改动清单：
- 圆角: `rounded-[28px]` → `rounded-lg` (8px)
- 背景: 渐变 → 纯白 `bg-white`
- 阴影: 极浅 → `0 2px 8px rgba(0,0,0,0.04)`
- 图片容器: 移除渐变背景
- 价格: 使用淘宝橙红色 `#ff4400` / `#ff5000`，数字放大，`¥` 符号缩小
- 原价: 划线灰色展示，弱于销售价
- 标题: 最多两行，超出省略
- 标签: 展示 2-3 个促销标签，避免撑高卡片
- 销量与店铺: 使用次级文本一行展示
- 卡片 hover: 轻微上浮或图片缩放，不使用厚重阴影
- 移除后台系统感的宽松 padding 和独立大 CTA，使卡片更接近商品流

### CTA 按钮体系

| 变体 | 样式 | 用途 |
|------|------|------|
| primary | 渐变暖橙底 + 白字 + 粗体 + 全宽 + 24px 圆角 | 购买、马上抢、去结算 |
| secondary | 橙边框 + 橙字 + 白底 | 加入购物车、查看更多 |
| ghost | 无边框 + 灰字 | 取消、返回 |

按压态: `scale(0.97)` + 暖橙阴影

### 商品详情页 `/products/[productId]`

**文件**: `product-detail.tsx` + `add-to-cart-panel.tsx`

- 价格独立成区，红色大字 32px
- SKU 选择按钮加大（44px 最小触控）
- CTA 双按钮堆叠（加购 + 马上抢）
- "马上抢" 按钮行为：先调用加购 API，成功后 `router.push("/cart")` 跳转购物车页

### 购物车页 `/cart`

**文件**: `cart/page.tsx` + `cart-list.tsx` + `cart-summary.tsx`

- 右侧汇总侧栏 → 底部固定结算栏（移动端优先）
- 条目减少卡片感，偏向列表
- 底部结算栏：全选 + 合计 + 去结算按钮

### Admin 后台页面 `/admin/*`

**不做运营风格改造**。后台需要高对比度、高信息密度，不适合暖色促销氛围。

- 背景：使用 `--bg-admin`（纯白 `#ffffff`），不用 `--bg-page`（灰白）或 `--bg-warm`（暖白）
- 字体：继承全局 `--font-body`
- 色彩：全局 CSS 变量自动生效（按钮变暖橙），这是可接受的副作用
- 布局：AdminShell 保持现有侧边栏 + 内容区结构不变
- 信息密度：保持高密度表格/列表布局，不降低密度

### 其他前台页面

- 结算汇总 `/checkout-summary`：更新色彩和按钮风格
- 下单确认 `/checkout`：更新按钮和色彩，底部固定提交
- 订单中心/详情 `/orders`：统一色彩和字体

## Constraints

- 保持 Next.js App Router + TailwindCSS v4
- 不改变任何 API 路由或数据流
- 优先修改 CSS 变量和组件样式，不动业务逻辑
- Server Component 优先（除非需客户端交互）
- 响应式：移动端为第一目标
- 不引入新的第三方 UI 或动效库
