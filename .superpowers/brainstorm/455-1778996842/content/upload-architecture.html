图片上传到阿里云 OSS，选哪种架构？

这是最核心的决策，决定了后端复杂度、安全模型和用户体验

方案 A：服务端中转上传
浏览器
选文件 → FormData
⬇️
后端 Controller
MultipartFile 接收 → 校验 → 调用 OSS SDK
⬇️
阿里云 OSS
PutObject → 返回 URL

优点

  • 实现简单，传统模式
  • 后端完全控制文件校验
  • 不暴露 OSS 凭证

缺点

  • 后端带宽压力大
  • 大文件占用内存/磁盘
  • 上传慢（两次传输）
方案 B：客户端直传 OSS（STS 临时凭证）
浏览器
1. 请求 STS 令牌
⬆️⬇️
后端 STS 接口
返回临时 AccessKey + Policy
⬇️
浏览器直接上传
2. OSS SDK JS → multipart 直传
⬇️
OSS 回调后端
3. 通知上传完成 + 校验

优点

  • 速度快，不占后端带宽
  • 支持分片/断点续传
  • 后端无压力

缺点

  • 需要 STS 服务
  • 前端稍复杂
  • 需配置 CORS/回调

推荐：方案 B — 客户端直传 + STS

对电商场景（多图、大图），客户端直传是业界标准做法。后端只负责签发临时凭证和接收回调，不参与文件流传输，性能最优。
