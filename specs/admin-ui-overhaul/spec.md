# Admin 后台界面改版规格

## 范围

三个区域的改造：首页运营合并、商品列表紧凑表格化、新建商品交互优化。

---

## 一、首页运营合并

### 现状

- `/admin/homepage` 只管 4 张活动卡片（2×2 运营位）
- `/admin/categories` 单独管理 10 个固定分类的排序和启停
- 管理员需在两个页面之间切换，认知负担大

### 目标

将分类导航管理合并到 `/admin/homepage`，同一页面管理首页全部内容。

### 布局

```
首页运营
├── 分类导航表格（10 行，紧凑表格）
│   ├── 分类名称（只读）
│   ├── 排序（inline 编辑）
│   ├── 状态（inline 编辑：启用/停用 toggle）
│   └── 编辑按钮 → 行内展开编辑区
├── 活动卡片 2×2 网格（保持现有交互）
└── 各自独立保存按钮
```

### 变化

- 分类以紧凑 `<table>` 展示，不再每行一张大圆角卡片
- "分类管理"从侧边栏导航移除
- 分类表格按 `sortOrder` 排序，与前台侧边栏一致

---

## 二、商品列表紧凑表格化

### 现状

- 每件商品渲染为一张大圆角卡片（封面图 120px + 信息区 + 操作按钮区）
- 筛选区是另一张大圆角卡片
- 大量空间浪费在圆角、阴影、内边距上
- 没有分页

### 目标

从卡片改为紧凑 `<table>`，参考订单表格 `admin-order-table.tsx` 的风格。

### 布局

```
商品管理                              [＋ 新建商品]
──────────────────────────────────────────────────
搜索: [关键词/SPU____] 分类: [全部▾] 状态: [全部▾] [查询]

共 128 个                        第 1/13 页
┌──────────────────────────────────────────────────────────────┐
│ 商品名称        SPU编码    分类     最低售价   状态    操作   │
├──────────────────────────────────────────────────────────────┤
│ iPhone 15...   SPU-001   手机数码   ¥7,999   已上架  编辑...│
│ 纯棉T恤 白M    SPU-042   服饰鞋包   ¥129     草稿    编辑...│
│ ...                                                        │
└──────────────────────────────────────────────────────────────┘
[< 上一页] [下一页 >]
```

### 列定义

| 列 | 内容 |
|----|------|
| 商品名称 | 纯文本，可点击进入编辑 |
| SPU 编码 | 等宽小字 |
| 分类 | 分类名称 |
| 最低售价 | 右对齐，¥ 前缀 |
| 状态 | 小徽章：草稿/已上架/已下架 |
| 操作 | 编辑 / 上架下架 / 删除，紧凑排列 |

### 变化

- 移除封面图缩略图（表格中不需要）
- 筛选栏改为顶部单行工具栏，不再是一张独立卡片
- 加入简单分页（前端 + 后端同步支持）
- 删除确认改为更友好的方式（暂保留 `window.confirm`，后续可替换）

## 三、新建商品交互优化

### 术语变更

| 旧术语 | 新术语 | 原因 |
|--------|--------|------|
| 展示属性 | 商品参数 | 运营人员更易理解，对应用于详情页展示 |
| 销售属性 | 规格属性 | 明确表达"用于生成 SKU 的规格维度" |

### 整体布局

表单采用分区式布局，每个分区用紧凑标题分隔（不用大圆角卡片包裹）：

```
< 返回商品列表

新建商品

▸ 基础信息
  4 列网格：分类 / 商品名称 / SPU 编码 / 副标题

▸ 商品图片
  左：商品主图（多张，第一张为封面）  右：商品详情图

▸ 商品描述
  全宽 textarea

▸ 商品参数（选填）
  紧凑表格：参数名 / 参数值 / 排序 / 删除

▸ 规格属性（选填，最多 2 组）
  紧凑表格：规格名 / 规格值（逗号分隔） / 删除
  + 自动计算将生成的 SKU 数量
  + 修改前弹确认框

▸ SKU 列表
  批量设置栏 + 紧凑表格
```

### 底部固定操作栏

始终固定在视口底部，三个按钮：

| 按钮 | 行为 |
|------|------|
| 取消 | 返回商品列表，不保存 |
| 保存草稿 | 保存为 `DRAFT`，可在商品列表中看到 |
| 保存并上架 | 保存为 `ON_SHELF`，前台立即可见 |

**不再在表单中放置"商品状态"下拉框**，状态由按钮决定。

编辑已有商品时底部按钮变为：

| 按钮 | 行为 |
|------|------|
| 取消 | 返回，不保存 |
| 保存修改 | 保持原状态不变 |
| 上架 / 下架 | 对应切换状态 |

### 分类下拉

- 下拉列表仅显示 `status=ENABLED` 的分类（已停用的不出现）
- 编辑已有商品时，如果其分类已被停用，仍显示但标记"已停用"且不可修改
- 辅助文案："仅可选择已启用分类"

### 商品主图

- 改为多张上传（和详情图组件功能一致），第一张自动作为封面
- 主图至少上传 1 张（必填校验）
- 建议尺寸 800×800

### 规格属性 → SKU 自动生成

**生成前弹确认框：**

> ⚠ 修改规格属性将重新生成 SKU
> 您已填写的部分 SKU 价格和库存数据可能丢失。
> 修改前 N 个 SKU → 修改后 M 个 SKU
> [取消] [确认重新生成]

**保留逻辑：** 如果旧的规格组合键（如 `颜色:黑|尺寸:S`）在新规格中仍存在，保留该行的价格、库存、阈值、状态。

**数量预估：** 实时显示"当前将生成 N 个 SKU（颜色 X 种 × 尺寸 Y 种）"

**数量过大提示：** 如果生成 SKU 超过 100 个，加一行提示："将生成超过 100 个 SKU，建议减少规格值数量。"

### SKU 批量填充

SKU 列表上方加批量操作栏：

```
批量设置： 价格 [_____]  库存 [_____]  阈值 [_____]  [应用到全部 SKU]
```

点击后覆盖所有 SKU 对应字段，已有数据的 SKU 也会被覆盖。

### 校验规则

| 字段 | 规则 |
|------|------|
| 分类 | 必选 |
| 商品名称 | 必填，2–100 字 |
| 商品主图 | 至少 1 张 |
| SKU 价格 | 必填，≥ 0，最多 2 位小数 |
| SKU 库存 | 必填，≥ 0 的整数 |
| 低库存阈值 | ≥ 0 的整数，允许大于库存（仅告警用） |
| SKU 编码 | 同一商品内不可重复 |
| 规格值 | 同一组内不可为空、不可重复 |
| 规格名 | 不可重复 |

## 不变的部分

- 后端 API 接口签名不变（前端适配层吸收差异）
- 数据库表结构不变
- 前台展示逻辑不变
- 10 个固定分类的硬编码列表不变（`homepage-categories.ts` 中的 `HOMEPAGE_CATEGORY_NAMES`）
- 活动卡片的 API 和数据模型不变
